# Sales & Delivery Workflow

## Discovery Call Framework (30-45 min)

### Opening (2 min)
```
"Thanks for taking the time. I'd love to understand what you're working on and see if there's a fit. I'll ask some questions, and at the end, I'll let you know honestly if I think I can help - and if not, I'll try to point you in the right direction. Sound good?"
```

### Problem Exploration (10 min)
**Questions:**
1. "Tell me about [problem area]. What's happening now?"
2. "What have you tried? What worked? What didn't?"
3. "What's the impact of this problem? On revenue? On time? On stress?"

### Desired Outcome (5 min)
**Questions:**
1. "If we solved this perfectly, what would that look like?"
2. "How would you know it was successful? What metrics matter?"

### Decision & Budget (5 min)
**Questions:**
1. "What's your timeline for tackling this?"
2. "What kind of budget have you allocated?"
3. "Who else needs to be involved in this decision?"

### Closing (5 min)
```
"Based on what you've shared, here's what I'm hearing..."
"I think [can/cannot] help. Here's why..."
"Next step would be [proposal/discovery deeper call/intro to someone else]"
```

**Always end with:**
```
"What questions do you have for me?"
```

## Proposal Framework (2-5 days)

### Structure
1. **Problem Recap** — In their words
2. **Proposed Approach** — Phased breakdown
3. **Deliverables** — Specific outputs
4. **Timeline** — Realistic milestones
5. **Investment** — Clear pricing
6. **Case Studies** — Similar work proof
7. **Scope** — What's included and not included
8. **Payment Terms** — 50% upfront recommended

### Payment Terms Options
- 50% upfront, 50% on delivery
- 33% upfront, 33% midpoint, 33% delivery
- Monthly retainer (for ongoing work)

## Negotiation Objections

| Objection | Response |
|-----------|----------|
| **Price too high** | "What part of the investment feels too high?" Often scope can adjust |
| **Timeline too long** | "What would need to change to make this timeline work?" |
| **Need approval** | "Who else should see this before we proceed?" |
| **Comparing to competitor** | "That's a fair comparison. Here's where we differ..." |
| **Want discount** | "I can adjust scope to fit budget, but quality stays fixed" |

## Project Milestones

### Phase 1: Discovery & Planning (Week 1-2)
- Deep dive requirements
- Technical architecture
- Wireframes/mockups
- Scope document finalized

### Phase 2: Core Development
- Build in 1-2 week sprints
- Weekly demos
- Continuous feedback
- Document as you build

### Phase 3: Review & Refinement
- Internal QA
- Client UAT
- Performance optimization
- Edge case handling

### Phase 4: Launch & Handoff
- Production deployment
- Monitoring setup
- Documentation delivery
- Training (if needed)
- Post-launch support (define duration)

## Communication Protocol

### Regular Updates
| Frequency | Type | Format |
|-----------|------|--------|
| Daily | Async update | 1-2 sentences in shared channel |
| Weekly | Progress email | Accomplishments, blockers, next steps |
| Bi-weekly | Video call | Complex projects only |
| Milestone | Demo session | Recorded walkthrough |

### Issue Escalation
| Severity | Response |
|----------|----------|
| Small blocker | Message in shared channel |
| Timeline risk | Email + call within 24 hours |
| Scope change | Written confirmation before proceeding |
| Major issue | Immediate call + written summary |

## Delivery Standards

| Standard | Requirement |
|----------|-------------|
| Code Quality | Clean, documented, maintainable |
| Security | No hardcoded secrets, proper auth, input validation |
| Performance | Load tested, optimized, error handling |
| Documentation | README, API docs, deployment guide |
| Testing | Unit tests for critical paths, integration tests |
| Handoff | All credentials, repos, docs in one folder |

## Pricing Strategies

### Fixed Price
- Best for: Well-defined projects with clear scope
- Pros: Easy for clients to budget
- Cons: Requires detailed discovery
- Rule: Add 20% buffer for unknowns

### Hourly Rate
- Best for: Ongoing or exploratory work
- Pros: Flexible for changing scope
- Cons: Can scare risk-averse clients
- Rule: Set weekly hour caps

### Retainer
- Best for: Ongoing relationships
- Pros: Predictable revenue, deeper relationships
- Cons: Need to define scope per month
- Rule: Review quarterly

## Red Flags

| Flag | Meaning | Action |
|------|---------|--------|
| "I need this yesterday" | Unrealistic timeline | Require detailed scope |
| "Can you work for equity?" | No budget | Decline unless exceptional |
| "I'll know it when I see it" | Scope creep | Force detailed requirements |
| "My last dev ghosted me" | May be difficult client | Get references |
| "It's a simple project" + vague | Never simple | Force specification |
| Won't share budget | Time-waster | Ask directly 2x, pass |
| Multiple decision makers | Approval nightmare | All parties on kickoff |
| NDA before discussion | Corporate bureaucracy | Standard NDA ok |
| Asking for free work | Doesn't value expertise | Decline politely |
| Price-focused only | Will nickel-and-dime | Fixed price with clear scope |
