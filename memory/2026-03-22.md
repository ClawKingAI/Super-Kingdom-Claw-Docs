### Agentic Design Patterns Study
**Repository:** https://github.com/evoiz/Agentic-Design-Patterns.git
**Book:** "Agentic Design Patterns: A Hands-On Guide to Building Intelligent Systems" by Antonio Gulli
**Location:** `/data/.openclaw/workspace/Agentic-Design-Patterns/`

#### 21 Core Patterns (organized by category)

**Part One: Core Patterns**
1. **Prompt Chaining** — Sequential task decomposition. Each step's output feeds into the next. Best for deterministic workflows where order matters. Use when tasks can be broken into clear steps.

2. **Routing** — Dynamic path selection based on input classification. A coordinator routes requests to specialist sub-agents. Uses RunnableBranch or conditional logic. Key for multi-domain agents.

3. **Parallelization** — Concurrent processing of independent tasks. Run multiple sub-agents simultaneously and merge results. Use when tasks don't depend on each other.

4. **Reflection** — Self-improvement through iterative critique. Generate → Reflect → Refine loop. The agent evaluates its own output against requirements and improves. Stopping condition: "CODE_IS_PERFECT" or max iterations.

5. **Tool Use** — External capability integration. Agents call APIs, search, execute code. Frameworks: LangChain tools, CrewAI tools, Google ADK toolsets.

6. **Planning** — Strategic task management. Agent decomposes complex goals into executable steps. Deep Research API example for multi-step research workflows.

7. **Multi-Agent** — Collaborative systems. Sequential, parallel, or coordinator-worker patterns. CrewAI example: Researcher → Writer pipeline with context passing.

**Part Two: Advanced Patterns**
8. **Memory Management** — State persistence across conversations. Types: ConversationBufferMemory, InMemoryStore, vector-based semantic search. Key for long-running agents.

9. **Learning and Adaptation** — Dynamic improvement through feedback. OpenEvolve example — agents that improve their own prompts/instructions based on outcomes.

10. **Model Context Protocol (MCP)** — Standardized interfaces for agent-tool communication. FastMCP server/client pattern. Filesystem tools example.

11. **Goal Setting and Monitoring** — Objective tracking. Agents iterate toward goals with progress evaluation.

**Part Three: Production Patterns**
12. **Exception Handling and Recovery** — Robust error management. Fallback chains, retry logic, graceful degradation.

13. **Human-in-the-Loop** — Human-AI collaboration. Customer support example with approval gates and escalation.

14. **Knowledge Retrieval (RAG)** — Information access patterns. Google Search, VertexAI, LangChain retrievers. Grounding responses in external data.

**Part Four: Enterprise Patterns**
15. **Inter-Agent Communication (A2A)** — Agent networking. AgentCard protocol for capability discovery. Calendar agent example with OAuth.

16. **Resource-Aware Optimization** — Efficient resource usage. Cost tracking, token management, parallel execution tradeoffs.

17. **Reasoning Techniques** — Advanced decision-making. Chain-of-Thought prompting, self-correction, code execution for math. Multi-step analytical thinking.

18. **Guardrails/Safety Patterns** — Risk mitigation. LLM-as-judge content policy. Pydantic validation. CrewAI guardrail functions. Policy enforcer pattern for input filtering.

19. **Evaluation and Monitoring** — Performance tracking. LLM-as-Judge for response quality. Basic response evaluation metrics.

20. **Prioritization** — Task management. SuperSimplePM example — agents that prioritize competing tasks.

21. **Exploration and Discovery** — Autonomous learning. Agent Laboratory example — agents that discover and explore new information.

#### Key Implementation Frameworks
- **LangChain/LangGraph** — Chain composition, memory, tool use
- **Google ADK** — Agent orchestration, toolsets, A2A
- **CrewAI** — Multi-agent workflows, role-based agents
- **FastMCP** — Model Context Protocol servers

#### Critical Insight
The book organizes patterns from foundational (Part 1) to enterprise (Part 4). For building production agents:
1. Start with core patterns (chaining, routing, parallelization)
2. Add reflection for quality improvement
3. Implement guardrails before production
4. Use evaluation loops for monitoring
5. Layer memory for stateful behavior
