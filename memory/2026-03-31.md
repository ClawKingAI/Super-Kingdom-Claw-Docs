### 2. OpenSpace Self-Evolution Engine

**Three Evolution Types:**
- **FIX** — Repair broken/outdated instructions (in-place)
- **DERIVED** — Create enhanced version from existing skill
- **CAPTURED** — Capture novel reusable pattern from execution

**Three Trigger Sources:**
1. Post-analysis — analyzer found evolution suggestions
2. Tool degradation — quality manager detected issues
3. Metric monitor — periodic scan of skill health

**Key Components:**
- `SkillRegistry` — Track skill metadata and lineage
- `SkillStore` — Persist skills to cloud/local
- `ExecutionAnalyzer` — Analyze task execution for patterns
- `SkillEvolver` — Execute evolution actions

**Results:**
- 46% fewer tokens through skill reuse
- 4.2× better performance on GDPVal benchmark
- Skills auto-fix when tools/APIs change
