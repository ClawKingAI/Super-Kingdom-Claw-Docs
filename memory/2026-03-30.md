### Implementation Created

#### 1. Skill: `self-evolution-engine`
**Location:** `/data/.openclaw/workspace/skills/self-evolution-engine/`

**Files:**
- `SKILL.md` — Architecture documentation
- `evolution_engine.py` — Core implementation

**Components:**

| Component | Description |
|-----------|-------------|
| `SkillStore` | SQLite persistence for skill records, lineage, metrics |
| `SkillRegistry` | Discovery, loading, matching, injection |
| `EvolutionEngine` | FIX, DERIVED, CAPTURED evolution operations |
| `QualityMonitor` | Metric thresholds, health checks |
| `SelfEvolutionEngine` | Main interface |

#### 2. Database Schema
**Location:** `/data/.openclaw/workspace/.openspace/schema.sql`

**Tables:**
- `skill_records` — Identity + lineage + quality
- `skill_lineage_parents` — Version DAG edges
- `execution_analyses` — One per task
- `skill_judgments` — Per-skill assessment
- `evolution_suggestions` — Pending evolutions
- `skill_tool_deps` — Tool dependencies
- `tool_quality` — Tool success rates
- `skill_search` — FTS5 virtual table

#### 3. Evolution Types

| Type | Description | Trigger |
|------|-------------|---------|
| FIX | In-place repair | Skill failed, tool changed |
| DERIVED | Enhanced version | Successful improvement detected |
| CAPTURED | Brand new skill | Novel pattern discovered |

#### 4. Quality Metrics

| Metric | Formula | Low Threshold |
|--------|---------|---------------|
| `applied_rate` | applied / selections | < 50% |
| `completion_rate` | completions / applied | < 60% |
| `fallback_rate` | fallbacks / selections | > 30% |
