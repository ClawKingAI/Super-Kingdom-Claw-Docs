## Hermes Agent Deep Dive: Learnings

### Source Repository
`/data/.openclaw/workspace/hermes-agent/` — Cloned from NousResearch

### Key Architectural Patterns

#### 1. Self-Improving Learning Loop
**Files:** `tools/skills_tool.py`, `agent/prompt_builder.py`, `agent/skill_commands.py`

Hermes has a **closed learning loop**:
- After complex tasks (5+ tool calls), agent is prompted to save as a skill
- Skills are stored in `~/.hermes/skills/<name>/SKILL.md`
- Skills self-improve during use: when outdated/wrong, patch immediately
- Skills are indexed and injected into system prompt at session start

**My Takeaway for Kingdom Claw:**
- Current: SKILL.md loaded once, not actively maintained
- Improvement: Add "self-improve" mechanism — detect skill issues mid-use, auto-patch
- Add `skill_manage(action='patch')` pattern from Hermes

#### 2. Memory System: Frozen Snapshot Pattern
**File:** `tools/memory_tool.py`

Critical insight: **System prompt memory is frozen at session start.**

- MEMORY.md and USER.md loaded once at session start
- Mid-session writes update files on disk but DO NOT change system prompt
- This preserves prompt caching (prefix cache stability)
- Next session gets fresh snapshot

**My Takeaway for Kingdom Claw:**
- We currently inject MEMORY.md into every turn via system prompt
- Hermes pattern is cleaner: load once, frozen for session, tools update live state
- Tool responses show live state, system prompt shows frozen state
- This reduces token usage and stabilizes caching

#### 3. Context Compression (Anthropic-Style)
**File:** `agent/context_compressor.py`

Sophisticated compression when approaching context limit:
1. Prune old tool results (cheap, no LLM call)
2. Protect head messages (system prompt + first exchange)
3. Protect tail messages by token budget (most recent ~20K tokens)
4. Summarize middle turns with structured LLM prompt
5. Iterative summary updates (preserves info across multiple compactions)

**Structure:**
- Goal
- Progress
- Decisions
- Files modified
- Next Steps

**My Takeaway:**
- This is better than simple truncation
- Structured summaries preserve semantic content
- Could implement similar pattern for long conversations

#### 4. Session Search with FTS5
**File:** `hermes_state.py`

SQLite-backed session storage with FTS5 full-text search:
- All messages stored with content, role, timestamp
- FTS5 virtual table for fast text search
- Search across all past conversations
- Returns relevant excerpts, not full transcripts

**My Takeaway:**
- Hermes has `session_search` tool — searches past conversations
- Kingdom Claw could benefit from similar capability
- Enables "What did we decide about X last week?" queries

#### 5. Honcho Integration: User Modeling
**File:** `tools/honcho_tools.py`

Honcho provides **dialectic user modeling**:
- `honcho_context` — LLM-powered Q&A about user (highest cost)
- `honcho_search` — semantic search over stored context (fast)
- `honcho_profile` — peer card with structured facts (fastest)

Three-tier access pattern:
1. Quick facts → `honcho_profile`
2. Search specific topics → `honcho_search`
3. Synthesize answer → `honcho_context`

**My Takeaway:**
- Honcho builds a model of the user across sessions
- More sophisticated than static USER.md
- Could integrate similar pattern for personalized agent behavior

#### 6. Delegate Tool: Subagent Isolation
**File:** `tools/delegate_tool.py`

Clean subagent architecture:
- Each child gets fresh conversation (no parent history)
- Own task_id (own terminal session, file ops cache)
- Restricted toolset (blocked: delegate_task, clarify, memory, send_message, execute_code)
- Parent sees only summary result, not intermediate steps

**Max depth:** 2 (parent → child → grandchild rejected)
**Max concurrent:** 3 children

**My Takeaway:**
- Similar to our `sessions_spawn` pattern
- Hermes blocks certain tools from children (no recursive delegation, no user interaction)
- Could adopt similar blocked tool list

#### 7. Skills System Architecture
**File:** `tools/skills_tool.py`

Progressive disclosure (Anthropic-inspired):
1. **Metadata tier** — name (≤64 chars), description (≤1024 chars) shown in skills_list
2. **Full instructions tier** — loaded via skill_view when needed
3. **Linked files tier** — references, templates loaded on demand

Directory structure:
```
skills/
├── my-skill/
│   ├── SKILL.md           # Main instructions (required)
│   ├── references/        # Supporting documentation
│   ├── templates/         # Output templates
│   └── assets/            # Supplementary files
```

Frontmatter fields:
- `name`, `description` (required)
- `platforms` — restrict to specific OS (macos, linux, windows)
- `required_environment_variables` — env checks
- `setup.collect_secrets` — prompts for API keys
- `conditions` — conditional activation (fallback_for_toolsets, requires_tools)

**My Takeaway:**
- We use similar structure but lack conditional activation
- Hermes skills can auto-activate when required tools available
- Skills can be "fallbacks" for missing toolsets

#### 8. Prompt Builder: Context File Priority
**File:** `agent/prompt_builder.py`

Priority order (first match wins):
1. `.hermes.md` / `HERMES.md` (walk to git root)
2. `AGENTS.md` / `agents.md` (cwd only)
3. `CLAUDE.md` / `claude.md` (cwd only)
4. `.cursorrules` / `.cursor/rules/*.mdc` (cwd only)
5. `SOUL.md` from HERMES_HOME (always included)

Each capped at 20,000 chars with head/tail truncation.

**Security:** Scans for prompt injection patterns before loading.

**My Takeaway:**
- We use similar AGENTS.md + SOUL.md pattern
- Hermes has more context file types and priority chain
- Injection scanning is smart — should add to Kingdom Claw

#### 9. Cron Scheduler: Tick-Based Execution
**File:** `cron/scheduler.py`

- Gateway calls `tick()` every 60 seconds from background thread
- Checks for due jobs, runs them
- File-based lock prevents concurrent ticks
- Jobs can deliver to: local, origin chat, specific platform

**My Takeaway:**
- Similar to our `cron` tool but simpler architecture
- Hermes cron is more integrated with gateway

#### 10. Tool Registry Pattern
**File:** `tools/registry.py` (implied from model_tools.py)

Each tool self-registers:
```python
registry.register(
    name="example_tool",
    toolset="example",
    schema={...},
    handler=lambda args, **kw: ...,
    check_fn=check_requirements,
    requires_env=["API_KEY"],
)
```

**My Takeaway:**
- Clean separation: tool definition, schema, handler, requirements
- Tools are discovered by importing modules
- Registry centralizes dispatch

### Summary: Key Improvements for Kingdom Claw

1. **Self-improving skills** — detect issues mid-use, auto-patch
2. **Frozen memory snapshot** — load once per session, preserve caching
3. **Context compression** — structured summarization, not truncation
4. **Session search** — FTS5 across past conversations
5. **Skills conditional activation** — auto-load based on tool availability
6. **Injection scanning** — detect prompt injection in context files
7. **Tool registry pattern** — cleaner tool discovery and dispatch

### Not Adopting

- Honcho integration (external dependency, adds complexity)
- Profiles system (we use single workspace)
- Skin engine (visual customization, not relevant)
- Auxiliary client router (we use single provider)

## Implementations Completed (2026-03-29)

### 1. Context Scanner (Injection Protection)
**File:** `/data/.openclaw/workspace/security/context_scanner.py`

Security scanner for context files (SOUL.md, AGENTS.md, MEMORY.md, etc.):
- 17 threat patterns covering injection, exfiltration, hidden content
- Invisible unicode detection (zero-width chars, BOM, directional overrides)
- Severity levels: critical (block), high (warn)
- Safe loading function that returns blocked message on threat

Based on Hermes `tools/skills_guard.py` patterns.

### 2. Subagent Blocked Tools
**File:** `/data/.openclaw/workspace/config/subagent-blocked-tools.json`

Tools blocked from subagent spawning:
- `delegate_task`, `sessions_spawn` — no recursive delegation (critical)
- `clarify` — no user interaction from children (high)
- `memory` — no writes to shared MEMORY.md (high)
- `send_message`, `message` — no cross-platform side effects (high)
- `execute_code` — children should reason step-by-step (medium)
- `tts` — no audio output (low)

Configuration: max_depth=2, max_concurrent=3

Based on Hermes `tools/delegate_tool.py` DELEGATE_BLOCKED_TOOLS.

### 3. Self-Improving Skills
**File:** `/data/.openclaw/workspace/skills/skill-patch-utils/SKILL.md`

Patterns for skill auto-patching:
- When to patch (outdated instructions, OS failures, missing pitfalls)
- Using `skill_manage(action='patch')` for targeted fixes
- Using `skill_manage(action='edit')` for full rewrites
- Patch logging format with timestamps
- Verification after patch

Based on Hermes `tools/skill_manager_tool.py` patterns.
